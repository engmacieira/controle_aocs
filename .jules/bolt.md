## 2026-07-21 - Dynamic imports overridden by manualChunks pattern **Learning:** Using dynamic `import()` for lazy-loading (such as `await import('xlsx')`) is an excellent way to reduce initial page bundle sizes. However, if the project has a custom `manualChunks` configuration in `vite.config.ts` that greedily bundles any `node_modules` import into a generic `'vendor'` chunk, the dynamically imported module will still get bundled into the main vendor chunk and preloaded by Vite. **Action:** When implementing lazy-loading / dynamic imports for vendor libraries, always check `vite.config.ts` and ensure the library is either excluded from manual vendor chunking or mapped to its own separate dynamic chunk, so that Vite does not preload it on initial load.
